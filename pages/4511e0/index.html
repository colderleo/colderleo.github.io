# getUserInfo

一个button的 openType="getUserInfo"时，同时绑定函数 bindgetuserinfo="authorLogin"，当用户点击该按钮会弹出授权提示，同意后会调用authorLogin。

wxml：

authorLogin(e){

console.log(e);

}

得到的e：

其中e.detail.rawData就包含了用户信息，就是以前的user_info

e.detail.userInfo也是用户信息。

不同的是e.detail.rawData是一个字符串，而e.detail.userInfo仍然是json结构体。

昵称，e.detail.userInfo.nickName，

头像，e.detail.userInfo.avatarUrl，

wx.getUserInfo调用时，success的返回结果记为res

//得到用户信息

​ wx.getUserInfo({

​ success: function (res) {

​ console.log(res);

​ },

​ },

res.rawData 和 e.detail.rawData 是一样的

注意这里的setStorageSync是set的字符串，读取的时候要JSON.parse解析成json结构体。

wx.setStorageSync('userInfo', e.detail.rawData);

var userinfo = JSON.parse(wx.getStorageSync('userinfo'));

# wx.login返回数据

小程序前端向微信服务器发送login，得到code，然后将code发送给小程序后台服务器 后台服务器向微信服务器发送http请求，包括code，appID和secret，成功后返回：

$wxResult
{
"session_key":    "9BC5cIgGo5nFRZmlvt3ENQ==",
"openid":     "oNqWP4sTyAJmVOZmOmgIc4IKt4P4"
}

失败可能返回：
{
"errcode":40029,
"errmsg":"invalid code, 
hints: [ req_id: xhm6304862160 ]"
}
也可能返回空或者其他的。

# wx.navigateTo，switchTab，redirectTo

navigateTo 和redirectTo 不能跳转到底部的几个主页面上。
https://developers.weixin.qq.com/miniprogram/dev/api/wx.redirectTo.html


wx.navigateTo({
          url: '../login/login'
        });

要用这个
    wx.switchTab({
      url: '../home/home',
    })



多个逗号不会跳转：
    // wx.navigateTo({
    //   url: 'pages/login/login',
    // });

要这样的：
    wx.navigateTo({
      url: '/pages/login/login'
    });

// url: '/pages/index/index',如果开头有斜杠，则是以根目录为参考路径，如果没有则是以当前目录为参考。
